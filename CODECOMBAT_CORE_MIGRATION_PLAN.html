Ke hoach chi tiet de nang todaycode-game-engine thanh core world/action/stateful simulation theo tinh than CodeCombat, sau do refactor todaycode-farmwars va todaycode-missions dua tren core moi. Tai lieu nay duoc viet nhu handoff doc cho team khac co the tiep tuc thuc hien.

1. Tom tat dinh huong

Muc tieu khong phai copy CodeCombat nguyen xi. Muc tieu la dua todaycode-game-engine tu mot contract mong thanh mot programmable world kernel: world co entity song, action co validate/apply, query API doc state that, event/replay du de render, grading/objective doc state cuoi.

Dinh huong dung: core khong chua FarmWars hay Missions rule. Core chi cung cap primitives va lifecycle chung. FarmWars va Missions la rulebook/domain packages implement tren primitives do.
PackageVai tro sau migrationKhong nen lam
todaycode-game-engine Kernel: Engine contract, World, Entity, Component, Action, System, Selectors, Replay, Objective/Grading primitives, deterministic utilities. Khong them game-specific logic nhu FarmWars tree/gift hay Missions wizard story.
todaycode-farmwars Arena rulebook: actor/tree/gift/base/scoring/combat/debuff/respawn, bot wire protocol, replay reducer. Khong tu tao lai world/action/event primitives neu core da co.
todaycode-missions CodeCombat-like campaign rulebook: wizard, enemies, items, barriers, API selectors, objective/win conditions. Khong tiep tuc command-log thuan tuy; API phai doc state da validate.

3. Kien truc muc tieu cua core moi

Core moi nen giu contract cu de backward-compatible, dong thoi them lop API moi. Domain packages co the migrate dan dan: adapter truoc, refactor sau.

3.1 World State

export type ComponentType = string;

3.3 Action lifecycle

export interface Action<Payload = unknown> {
  type: string;
  actorId: EntityId;
  payload: Payload;
  source?: 'runner' | 'system' | 'auto';
}

export interface ActionHandler<Payload = unknown> {
  type: string;
  validate(ctx: ActionContext, action: Action<Payload>): ValidationResult;
  apply(ctx: ActionContext, action: Action<Payload>): ActionResult;
}

Action chung gom MOVE, ATTACK, PICK_UP, CAST, SAY, WAIT. Domain dang ky action rieng nhu FarmWars GIFT, BASE_SETUP, HARVEST, PLANT_TREE.

3.4 System pipeline

  • MovementSystem: bounds, collision, blocked event.
  • CombatSystem: range, damage, hp, death events.
  • CollectionSystem: item pickup, inventory, collect events.
  • CooldownSystem: decrement cooldown deterministic.
  • ObjectiveSystem: evaluate conditions after action/tick.
  • ScoringSystem: domain-specific scoring.
  • DeathRespawnSystem: FarmWars respawn/debuff/death flow.

3.5 Selectors va programmable API

CodeCombat-like API phai doc tu state that sau moi action hop le. Core can co selectors game-agnostic, domain wrap thanh API hoc sinh.

6.4 API runner can bam vao core

Runner Python/JS khong nen chi emit command roi cuoi cung moi simulate. Với Missions, can interleaved action/query flow neu muon giong CodeCombat:

  1. User code goi wizard.move_right().
  2. Runtime gui action MOVE sang engine host.
  3. Engine validate/apply action, update world.
  4. Runtime tiep tuc; neu user goi find_nearest_enemy(), query doc world da update.
  5. Stop khi code ket thuc, win, fail, timeout, hoac max actions.
Migration thuc te: neu interleaved runner qua lon, phase dau co the van batch actions nhung phai chuyen query APIs sang state chinh xac nhat co the. Phase sau moi lam host bridge interleaved.

  • Scoring: harvest deposit, worm steal, gift reward unchanged.
  • Concurrency irrelevance: engine pure, no hidden singleton state between matches.

8. Roadmap thuc hien

Phase 0 - Freeze behavior hien tai

  • Chay va luu baseline tests cho 3 package.
  • Chot golden fixtures FarmWars va Missions truoc khi refactor.
  • Ghi ro package versions dang dung trong BE/FE.

Phase 1 - Core additive API

  • Them world/entity/components/actions/selectors/events vao todaycode-game-engine.
  • Khong sua breaking Engine.
  • Release @amas-nghia/todaycode-game-engine@0.3.0.

Phase 2 - Missions proof

  • Bump game-engine dependency trong todaycode-missions.
  • Refactor state/order/tick/selectors theo core primitives.
  • Implement combat/objectives day du cho CS1 Ch.1-Ch.2.
  • Release todaycode-missions@0.2.0.

Phase 3 - FarmWars adapter

  • Bump game-engine dependency trong todaycode-farmwars.
  • Them adapter state-to-world, world-to-state hoac migrate native state dan dan.
  • Chuan hoa event metadata cho replay.
  • Giu bot protocol va scoring unchanged.
  • Release todaycode-farmwars@0.4.0 hoac version tiep theo.

Phase 4 - Consumers

  • Bump dependencies trong todaycode-be.
  • Bump dependencies trong CodeClash-fe.
  • Cap nhat FE replay/mission runner neu replay v2 fields duoc dung.
  • Chay integration/e2e.

Phase 5 - Interleaved runner for Missions

  • Thiet ke host bridge cho user code: action/query/action/query.
  • Dam bao timeout/max actions/max statements.
  • Move query APIs sang state da validate hoan toan.
  • Day la phase gan CodeCombat nhat, nen lam sau khi core primitives on dinh.

9. Checklist ban giao

Core Definition of Done

  • Core co world/entity/component/action/selector/event APIs.
  • Khong import gi tu FarmWars/Missions vao core.
  • Determinism tests pass.
  • Old Engine implementations van compile.
  • README co migration notes.

Missions Definition of Done

  • CS1 movement/combat/pickup scenarios pass.
  • Selectors doc state that.
  • Win conditions khong throw cho supported schema.
  • Golden fixtures pass on both old-compatible and new core path.

FarmWars Definition of Done

  • Existing bot matches pass.
  • Replay final score/state reproduce duoc.
  • Event metadata du de render attack/damage/death.
  • Dynamic config behavior unchanged and tested.

Risk Can canh giac

  • Breaking package exports lam BE/FE fail build.
  • Iteration order khong deterministic.
  • Replay v2 khong backward-compatible voi replay cu.
  • FarmWars scoring drift trong luc refactor systems.
  • Missions runner batch vs interleaved semantics bi tron lan.

Quyet dinh kien truc nen chot truoc khi code

  1. Core se release additive 0.3.0 hay breaking release?
  2. WorldState dung entity component generic den muc nao, hay chi cung cap helpers?
  3. Replay v2 co bat buoc keyframe/config trong core envelope khong?
  4. Missions phase dau van batch actions hay lam interleaved runner ngay?
  5. FarmWars co migrate native state sang WorldState hay dung adapter lau dai?
Recommendation: Lam additive core truoc, migrate Missions truoc, FarmWars sau. Day la duong it rui ro nhat vi Missions nho hon va gan CodeCombat hon; FarmWars co scoring/replay phuc tap nen can golden fixtures bao ve truoc khi tach systems.
